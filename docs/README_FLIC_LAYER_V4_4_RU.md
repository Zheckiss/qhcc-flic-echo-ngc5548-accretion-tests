# QHCC / FLiC NGC 5548 layer test v4.4

## Что это

`qhcc_ngc5548_flic_layer_test_v4_4.py` — это отдельный диагностический скрипт для формальной проверки структуры, которая видна на графике “задержка — длина волны”.

Скрипт **не ищет новые пики** и **не меняет результаты v4**. Он берёт уже полученные таблицы:

- `flic_candidates_ranked_v4.csv`
- `baseline_summary_v4.csv`
- `ngc5548_flic_delays_v4.csv`
- `null_pvalues_combined_v4.csv`

и проверяет, насколько наблюдаемые лаги организованы вокруг двух фиксированных FLiC-шкал:

- одноходовая ветвь: `one_way_2over3`;
- двухходовая ветвь: `two_way_4over3`.

## Главная проверка

Скрипт считает остатки одноходовых точек относительно фиксированной задержки

\[
\Delta t_{2/3}
\]

и проверяет:

1. насколько мал разброс вокруг одноходовой FLiC-линии;
2. лежат ли коротковолновые каналы ниже одноходовой линии;
3. лежат ли длинноволновые каналы выше одноходовой линии;
4. лучше ли FLiC-линия организует точки, чем простая дисковая кривая;
5. насколько редко случайная горизонтальная линия даёт такой же малый разброс;
6. насколько точно главный двухходовой кандидат `opt_V_daily.csv` совпадает с `two_way_4over3`.

## Команда запуска

Из папки проекта:

```bat
python qhcc_ngc5548_flic_layer_test_v4_4.py ^
  --ranked run_v4\results\flic_candidates_ranked_v4.csv ^
  --baseline run_v4\results\baseline_summary_v4.csv ^
  --delays run_v4\results\ngc5548_flic_delays_v4.csv ^
  --null-pvalues run_v4\results\null_pvalues_combined_v4.csv ^
  --out-dir run_v4\results
```

## Как читать результат

Главные числа в `flic_one_way_layer_summary_v4_4.csv`:

- `rms_delta_to_one_way_hours` — средний квадратичный разброс вокруг одноходовой FLiC-линии;
- `mad_abs_delta_to_one_way_hours` — среднее абсолютное отклонение;
- `mean_delta_short_wavelength_hours` — среднее смещение коротковолновых каналов;
- `mean_delta_long_wavelength_hours` — среднее смещение длинноволновых каналов;
- `disk_fixed_rms_hours` — разброс относительно простой дисковой кривой с beta=4/3;
- `disk_free_rms_hours` — разброс относительно дисковой кривой со свободной beta;
- `random_line_p_as_good_as_one_way` — доля случайных горизонтальных лагов, которые организуют точки не хуже одноходовой FLiC-линии.

## Текущий результат на приложенных таблицах

Для текущего набора NGC 5548:

- одноходовая FLiC-задержка: около 12.80 ч;
- двухходовая FLiC-задержка: около 25.59 ч;
- одноходовый слой содержит 8 точек;
- RMS вокруг одноходовой FLiC-линии: около 1.88 ч;
- коротковолновые каналы лежат в среднем ниже одноходовой линии;
- длинноволновые каналы лежат в среднем выше одноходовой линии;
- простая дисковая кривая даёт больший RMS;
- случайная горизонтальная линия даёт такой же или меньший RMS примерно в 1.5% случаев;
- главный двухходовой кандидат `opt_V_daily.csv` даёт 25.44 ч против ожидаемых 25.59 ч.

## Важная оговорка

Это структурная диагностика, а не окончательная глобальная статистика открытия. Она показывает, что наблюдаемые лаги организованы относительно заранее фиксированных FLiC-шкал лучше, чем относительно простой дисковой кривой или типичной случайной горизонтальной линии.
